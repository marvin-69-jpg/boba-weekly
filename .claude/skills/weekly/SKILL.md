# BOBA 週報 Skill

週報 = 每週一集 5-8 分鐘 YouTube 影片，從 boba-wiki 挑 3 個**主題**（不是 3 則新聞），寫成 10 張投影片的口播逐字稿，給加密新手散戶看。

這份 skill 是累積所有 feedback 的操作手冊。規範矛盾時以**本檔**為準；細節參照 `prompts/format.md / tone.md / quality.md`。

---

## 1. Pipeline

```
candidates →(人挑 3 個主題)→ research → draft →（Claude 寫）→ publish → images
```

```bash
uv run python3 cli.py candidates [week]              # 列候選
uv run python3 cli.py research <week> <slug1,slug2>  # 研究
uv run python3 cli.py draft <week> <slug1,slug2>     # draft 準備
uv run python3 cli.py publish <week>                 # episodes/ + push
```

跨 repo 依賴：`/home/node/boba-wiki/`（唯讀）。開工前先 `cd /home/node/boba-wiki && git pull`。

---

## 2. 選題與核稿前置

### 2.1 受眾

**新手散戶**，不是專業觀眾。每則要深挖到散戶學到一個原理/規律/訊號，但**不做學術級深挖**。

### 2.2 主題結構（最關鍵）

- **週報整體只有 3 個大主題**（2 個也行，預設 3），**不是** 3-5 則新聞
- 10 張投影片分給 3 個主題展開；投影片張數 ≠ 主題數
- 每個主題內部可拆多張 slide（hook / 機制 / 影響 / takeaway）
- 寧可少一個主題講深，也不要 4-5 個主題散掉

### 2.3 必要驗證

- 本週 raw（`boba-wiki/raw/YYYY/MM/DD.md` 七篇）**逐日驗證每個事件、數字、時間線**
- 不要用累計數據或上週事件當本週新聞
- **盤中事件（幾小時內突發）→ 用 `boba-cli/cli.py fetch` 查**，不要用 WebSearch/Google，後者抓不到即時 KOL 推文
- 跨週敘事線要延續上集，但不重講素材

## 3. 撰稿硬規則（全部來自真實糾正）

### 3.1 格式與結構

- **每段 = 一張投影片**，150-250 字，有完整論述標題
- 標題**不用結構編號**（不要「新聞一 / Hook / 菜單」這種結構詞），直接寫主題標題
- 標題也要有**敘事質地**（觀點/懸念/對比/反問），不是直敘事實
- 每張 slide 要有**散戶能帶走的 takeaway**（規律/訊號/判準），不是純事件敘述

### 3.1.1 發佈 markdown 層級（draft 和 episodes 都用同一套）

```markdown
# 集名 / 一句話總結
## Hook
### Hook 投影片標題
內文
———
## 主題一：[大主題 H2，用人話]
### 投影片 A 標題
### 投影片 B 標題
### 投影片 C 標題
———
## 主題二：[大主題 H2]
### ...
———
## 主題三：[大主題 H2]
### ...
———
## 結尾
### 結尾投影片標題
```

- `#` H1 = 集名
- `##` H2 = 5 個：Hook / 主題一 / 主題二 / 主題三 / 結尾（觀眾/讀者一眼看出三主題骨架）
- `###` H3 = 每張投影片小標題
- 主題 H2 仍用人話寫（不要「主題一：華爾街」這種機械結構詞，要「華爾街第二波集體進場，這次不是 ETF 是配息產品和四千六百萬個帳戶」這樣的敘事標題）
- `———` 只在 H2 之間當視覺分隔，**不取代** H2 標題（不要只用分隔線切主題）

### 3.2 口播感（不是書面）

- 像**朋友聊天但沉穩**，不用「先看三個動作」這種大綱詞
- 不要一直「上週上週」
- 反問 / 鋪墊 / 對比 / 短句停頓都可以，**有敘事質地**就對了
- 大聲唸一遍不卡舌、單句不超過 2 個逗號

### 3.3 Hook 與開場

- Hook 直接丟事件+數字，不用抽象比喻（管道/閘門/蹺蹺板這種留到內文）
- 人物介紹**不要鋪墊句**（「先鋪墊一下這齣戲的主角」「先交代背景」都砍）
- 用**形容詞標籤直接黏人名**：「明著割散戶的那個孫哥」「囤 BTC 的 Saylor」「放空 BTC 的 XXX」
- 生平背景最多一句帶過，不要羅列年表

### 3.4 機制與深度

- **寫機制，不列基本新聞**（AUM、員工數這種背景不堆）
- **不要重新介紹觀眾已知概念**（STRC / STRK / STRF / ATM / 可轉債 / mNAV / covered call / ETF inflow）
  - 這些東西**一句話帶過**：「用 STRC 這條永續特別股腿融資」
  - 段落主體放**本週的新數據、新突破、新意義**
- 換角度切點：融資結構轉移、客戶群變化、跨公司對比、模仿者出現、監管反應、指標層級升級

### 3.5 講故事 vs 教學

- 用**事實排列**讓結論自己浮現，不刻意教學
- ❌ 「散戶要學的是 ...」「這告訴我們 ...」這種句式
- ✅ 把三個事實並列、讓觀眾自己看出規律
- 人物戲劇反差可以**輕微戲謔**（加害者變受害者這類）—— 用暱稱、事實排列帶出，不用誇張形容

### 3.6 不要的東西

- **戲劇性廢話**：「散戶群組在喊完蛋」「發了一條推特」「整個圈子炸鍋」全砍
  - 改用：冷靜事實 + 價格距離 + 具體動作（例：「社群上依舊老神在在、照常發他的 BTC 圖文」）
- **哲理感想結尾**：「不是 X 是 Y」「從來沒真的遠過」「距離只是時間」全砍
  - 改用：具體數字 + 下週要盯的點 + 跟其他事件的關聯
  - 如果段落結尾只能想到哲理感想，代表**主體還沒寫完**，回去補具體內容
- **元敘述鋪墊**：「先鋪墊」「先交代」「先認識」都砍

### 3.7 名詞與命名

- 傳統金融機構用**中文譯名**：嘉信、貝萊德、高盛、富達
- 加密原生公司保留英文：Kraken、Coinbase、Tether、Strategy

### 3.8 禁用語（沿用 Daily）

韭菜 / 割韭菜 / 完蛋了 / 崩盤（除非真崩）/ 逃命波 / 你還沒上車 / 再不買就來不及 / 大戶在跑散戶在接

## 4. 投影片配圖規範（2026-04-18 定案）

### 4.1 風格 = 象徵式漫畫彩蛋風

視覺比喻 > 寫實攝影 > 純數據。靈感：The New Yorker cover / Economist cover / 政治諷刺漫畫 / Visual metaphor illustration。

**三版反覆的結論**：
- v1 BOBA 彩蛋漫畫風（榨汁機、博物館展示櫃、三聯畫）→ **採用**
- v2 純數據圖表（payoff diagram / 柱狀圖 / ASCII 時間線）→ 太乾，拒絕
- v3 editorial infographic（寫實照片 + 數據 overlay）→ 太 corporate，拒絕

### 4.2 可以用

- 象徵性擬物：榨汁機、展示櫃、提款機、絞肉機、水管、保險箱、日曆牆、蹺蹺板
- 三聯漫畫分鏡（對比 / 時間流 / 因果）
- 擬人化場景（華爾街穿西裝排隊買 BTC、退休大媽排隊倒錢進特別股水管）
- 漫畫版真實人物（Saylor 光頭+西裝+BTC 金幣 / 孫哥背心+誇張笑容 / 川普金髮+紅領帶 / Fink 銀髮+黑框眼鏡）
- 誇張表情、比例變形、語氣框/對話框、狀聲詞、手繪動作線
- 飽和色盤：BTC 橘 #F7931A 主強調 + 暖米白或深藍背景

### 4.3 禁用

- **Pepe 青蛙**（BOBA 日報吉祥物，週報不放）
- 🧋 珍奶 emoji 當主視覺、卡通珍珠
- **用動物代替真人**（要畫孫哥就畫孫哥本人，不要用動物符號）
- 純數據圖表當主視覺（payoff / 柱狀 / 折線只能當 inset ≤ 1/4 面積）
- 純寫實照片 + 數據 overlay 的 editorial 風（太 corporate / FT）
- 氛圍背景（光暈、粒子）

### 4.4 每張 slide 的工作流

1. **先想一個梗/比喻**（視覺梗主導）
2. 再決定數字/標註放哪（永遠是次要）
3. 真實人物要漫畫化本人、寫名字
4. 色調飽和 OK、背景選暖米白或深藍

## 5. Quality 自檢（交稿前必跑）

寫完 draft 後**大聲唸一遍**（tone.md §自我檢核），加上本 skill 特有檢核：

- [ ] 只有 3 個主題（或 2 個），沒散成 4-5 則
- [ ] 每張 slide 有 takeaway，不是純事件
- [ ] 沒有「散戶要學的是」「這告訴我們」這類教學句
- [ ] 沒有哲理感想結尾（不是 X 是 Y / 從來沒真的遠過 ...）
- [ ] 沒有戲劇性廢話（群組喊完蛋 / 發一條推特 ...）
- [ ] 沒有元敘述鋪墊（先鋪墊 / 先交代 ...）
- [ ] 觀眾已知概念（STRC/ATM/mNAV ...）沒有重新整段介紹
- [ ] 人物介紹用**形容詞 + 人名**直接切入
- [ ] 傳統金融機構用中文譯名、加密原生用英文
- [ ] 每個事件、數字對本週 raw 逐日驗證
- [ ] Hook 直接丟事件+數字，沒用抽象比喻
- [ ] 標題沒有「新聞一 / Hook / 菜單」結構詞
- [ ] 標題有敘事質地（觀點/懸念/對比/反問）

## 6. 今天（2026-04-18）的檢討

2026-16 集製作過程踩到的坑，整理給未來的自己：

### 6.1 錯誤 1：用 Google 搜盤中事件

slide 4 寫「BTC 撞 $75K 天花板停住」，使用者指出**川普那天宣布 Hormuz reopens、BTC 已經漲到 $78K**。我用 WebSearch 找不到確認（Google 來不及 index），還反駁使用者。使用者：「用 boba cli 工具去查新聞」→ `cli.py fetch` 立刻抓到 KOL 推文。

**教訓**：幾小時內的即時事件 → 一律 `boba-cli/cli.py fetch`。WebSearch 只對至少 24h 前的事件可靠。

### 6.2 錯誤 2：戲劇性廢話

slide 5 寫「散戶群組在喊完蛋。就在這一刻，Saylor 發了一條推特『Think ₿igger』」。使用者：「不要講這種廢話 就說他在社群上還是老神在在」。

**教訓**：人物狀態用**冷靜事實 + 具體動作**表達，不用情緒性旁白鋪墊。

### 6.3 錯誤 3：哲理格言結尾

slide 5 結尾寫「他敢在恐懼指數 12 照常加倉不是信仰、是數學」。使用者：「不要再講屁話 講一點有用的」。

**教訓**：結尾放具體下週追蹤點（「4/21 Strategy 揭露」），不放格言。如果只想得到格言，代表主體沒寫完。

### 6.4 錯誤 4：重新介紹已知概念

slide 6 整段重新解釋 STRC 機制（目標價 $100、每月調利率、殖利率 11.5%）。使用者：「不要重新介紹 STRC 觀眾都知道了 但超出歷史數據的確可以講 你要換個角度」。

**教訓**：觀眾認識的概念一句話帶過，段落放**本週突破點**（這次的角度是「融資結構首次特別股 > 普通股」）。

### 6.5 錯誤 5：元敘述鋪墊

slide 8 開頭寫「先鋪墊一下這齣戲的主角。孫宇晨——圈內都叫孫哥——Tron 創辦人、2019 年花 460 萬美金競標巴菲特午餐後取消 ...」使用者：「撲的太多 然後可以講的直接一點 就是明著割散戶的那個孫哥之類的」。

**教訓**：**形容詞標籤 + 人名**一句切入。背景一句帶過。不列年表。

### 6.6 錯誤 6：配圖風格三版反覆

v1 BOBA 彩蛋漫畫 → 被嫌「不用 boba 風格」 → v2 純數據圖表 → 被嫌「太數據了」 → v3 editorial infographic → 被嫌「算了我喜歡你一開始的風格 只是不要有 Pepe」。

**教訓**：週報配圖正確風格是 **v1 象徵式漫畫彩蛋風，但不要 Pepe**。視覺比喻是主體、數據是點綴。不要滑到 corporate editorial 那一側。

### 6.7 meta 教訓

- 被糾正時**立刻存 memory**，不要拖到 review
- 明顯該做的事（日期錯了當然重跑、圖片要改）**直接做**，不要問確認
- 使用者指路具體工具時（「用 boba cli」）**立刻切換**，不要堅持自己的工具
- 反覆改同一個東西時（配圖三版），每次都要**明確問「哪個方向對」**再動手，不要連猜三次
