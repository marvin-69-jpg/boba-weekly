# BOBA 週報品質自檢清單

> 初稿完成後逐項檢查。FAIL 項自動修復最多 2 次。

---

## 結構完整性(硬性門檻)

- [ ] 有 Hook 嗎?開頭 3 句內有抓住觀眾的數字或反直覺事實嗎?
- [ ] 有「本週菜單」嗎?預告 3-5 則新聞但不給答案?
- [ ] 新聞段數落在 **3-5 則**之間?
- [ ] 每則新聞有「事實 → 背景 → 拆解 → 過渡」四層嗎?
- [ ] 新聞之間有自然 transition 嗎?(不是「接下來看第二則」這種死板)
- [ ] 有結尾段嗎?給了 takeaway + 下週追什麼?
- [ ] 總長度 **1500-2500 中文字**(不含標點)?

## 數據與來源

- [ ] 每則新聞至少 **2 個具體數字**?
- [ ] 每個數字都有標來源(「根據 X 報導」「XX 官方公告」)?
- [ ] 來源是一手媒體 / 研究機構 / 官方數據?
- [ ] 所有數字是從 wiki 或研究 notes 確認的**最新**數據?(不是憑記憶)
- [ ] research.md 保留完整來源 URL?

## 口播感(逐字稿專屬)

- [ ] 大聲唸一遍,**卡舌的地方有拆**?
- [ ] 單句**不超過 2 個逗號**?三個以上一定要拆
- [ ] **沒有 Markdown 標題 `#`**?用自然語言過渡
- [ ] 段落**短**(3-5 句)?
- [ ] 重要數字放句尾方便停頓?
- [ ] 有沒有「其實」「的話」「這個部分」這種口語贅字?能砍就砍

## 語氣與風格

- [ ] 用「你」和「我」對話嗎?(不是「投資者」「觀眾」)
- [ ] 沒有 BOBA 禁用語?(見 tone.md)
- [ ] 術語第一次出現**即時解釋**?(「S-1 就是⋯⋯」)
- [ ] 壞消息有包得溫暖嗎?給方向感嗎?
- [ ] 沒有給明確投資建議(「應該買」「建議賣」)?
- [ ] 沒有製造 FOMO(「你還沒上車?」)?

## 開場與結尾

- [ ] 開場**不是**「大家好」「歡迎回到」這類平淡開頭?
- [ ] Hook 15 秒內丟出一個具體數字或反直覺事實?
- [ ] 結尾不是「以上是本週週報」官腔?
- [ ] 結尾**不硬塞**「訂閱按讚開啟小鈴鐺」?(這些影片剪輯時處理)
- [ ] 結尾有給下週追什麼具體指標 / 日期?

## Transition(週報的靈魂)

- [ ] 每則新聞結尾**有機地**接下一則,不是「接下來看⋯⋯」?
- [ ] Transition 建立新聞之間的**邏輯關聯**(對比、因果、同主題、跨主題)?
- [ ] 整集 3-5 則讀完,有一個**整體 insight**貫穿(結尾能點出來)?

## 分析深度(週報要比 daily 多一層)

- [ ] 每則不只是**列事實**?有拆解「為什麼」和「所以呢」?
- [ ] 有**跨主題連結**?(這件事跟另一件事的關聯,或跟 wiki 其他 entity)
- [ ] 有至少一個「欸,我沒想到」的 insight?
- [ ] 有用**比喻**把抽象數字變具體嗎?

## 格式

- [ ] 用 `———`(三個全形 dash)分隔新聞段?
- [ ] 沒有 Markdown `#` 標題?
- [ ] 數字寫法一致(USD 用 `$`,重要數字可以寫中文「一千億」)
- [ ] 收尾有 BOBA 感,溫暖不官腔?
