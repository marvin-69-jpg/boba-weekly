# BOBA 週報影片逐字稿結構

> 從 BOBA Daily / Deep Dive 的語氣拆解的 YouTube 週報模板。
> 語氣規範見 `tone.md`,品質自檢見 `quality.md`。

---

## 平台 + 規格

- **平台**:YouTube 中型影片(非 Shorts)
- **長度**:5-8 分鐘,**1500-2500 中文字**(不含標點)
- **週期**:ISO week(週一到週日),一週一集
- **聚合**:3-5 則新聞串場,不是流水帳

---

## 結構概覽

```
Hook(15 秒)
→ 本週菜單(20 秒)
→ 新聞 1(60-90 秒)
→ 新聞 2(60-90 秒)
→ 新聞 3(60-90 秒)
→ [選配:新聞 4, 5]
→ 結尾與下週預告(20-30 秒)
```

---

## 1. Hook(15 秒,80-120 字)

開場 15 秒決定觀眾留不留。**不要自我介紹,不要「大家好」**。

**模式(三選一)**:
- **反直覺事實**:「這週最炸的一件事,不是 XX,是 YY。」
- **大數字震撼**:「這週美股有一檔個股蒸發了 XX 億美元市值,你可能沒注意到。」
- **集體現象**:「你最近是不是常聽到 AI 泡沫這個詞?這週有四個訊號告訴我們⋯⋯」

**範例**:
> 這週科技圈最炸的不是輝達財報,是 OpenAI 悄悄簽了一張讓所有人下巴掉下來的晶片訂單。
> 1000 億美金。這是什麼概念?我們等下拆給你看。

**禁止**:
- ❌ 「大家好,歡迎回到 BOBA 週報」(太平)
- ❌ 先介紹自己 / 頻道
- ❌ 開頭就講「本週我們來聊」

---

## 2. 本週菜單(20 秒,100-150 字)

預告這集會講哪 3-5 則,給觀眾留下來的理由。

**模式**:
- 一則一行,**給懸念不給答案**
- 最後一句連回 Hook 或拋更大的問題

**範例**:
> 這集會幫你追這週三件事:
> OpenAI 那筆 1000 億晶片訂單到底怎麼算的、
> Bitcoin 跌破 9 萬美金背後誰在賣、
> 還有 Google 的新模型發佈為什麼這次大家特別不一樣。
> 好,我們從 OpenAI 開始。

---

## 3. 每則新聞段(60-90 秒,300-450 字)

這是肉。每則新聞要有三層:**發生什麼 → 為什麼重要 → 你該怎麼看**。

**每則的子結構**:

1. **事實(60-90 字)** — 這週發生了什麼,關鍵數字放前面
2. **背景(80-120 字)** — 這不是孤立事件,背景是⋯⋯(用 boba-wiki 的 Key Events 壓縮)
3. **拆解(100-150 字)** — 這個新聞的 insight,兩方觀點(如適用),或一個跨主題的連結
4. **過渡(30-50 字)** — 自然接到下一則

**範例開頭(事實)**:
> 先講數字。OpenAI 這週跟博通簽了一張 10GW 的客製化晶片訂單,規模估計超過 1000 億美元。消息出來當天博通股價跳 9%,一天吃掉 400 億美元市值。

**範例過渡**:
> OpenAI 用硬體綁生態,接下來看另一頭,市場開始擔心這整個 AI 投資在估值上有沒有透支。這週 Bitcoin 的跌法,說不定就是第一個訊號。

**規則**:
- 每則**至少 2 個具體數字**,都要標來源(「根據 X 報導」「XX 公告」)
- 不要寫成 bullet list —— 影片是口播,要連貫的段落
- 每則結尾必接下一則(transition 是週報的靈魂)

---

## 4. 結尾與下週預告(20-30 秒,120-180 字)

**不是總結 bullet**。是給觀眾一個 takeaway + 下週追什麼。

**模式**:
- 1-2 句本週整體 insight(跨越多則新聞的共同點)
- 1 個觀眾可以開始關注的指標 / 日期
- 不強拉 CTA,自然收尾

**範例**:
> 這週把三件事串起來看,你會發現一個重點:市場資金正在重新排序,AI 這一輪還不會結束,但估值已經開始分層。
> 下週要看的是 NVDA 財報跟美國 CPI,這兩個數字會決定市場敢不敢繼續追。
> 我們下週見。

**禁止**:
- ❌ 「以上是本週週報」(太官腔)
- ❌ 「別忘了訂閱按讚開啟小鈴鐺」(硬 CTA 放影片剪輯,不寫在逐字稿)
- ❌ 「如有投資請自行判斷」(用 BOBA 語氣自然處理,見 tone.md)

---

## 格式規則

| 規則 | 說明 |
|------|------|
| **無 Markdown 標題** | 逐字稿是口播用,不用 `#`,用自然語言過渡 |
| **段落短** | 一段 3-5 句,方便口播斷句 |
| **換行** | 新聞之間用 `———`(三個全形 dash) |
| **數字** | USD 用 `$`,原幣別保留,重要數字可以直接念出來(「三百六十億」而不只是 `$36B`) |
| **來源** | 行內標出:「根據 Bloomberg 的報導」「OpenAI 官方聲明」 |
| **連結** | 逐字稿不帶超連結,但 research.md 保留完整 URL |
| **專有名詞** | 第一次出現即時翻譯(見 tone.md「術語即時翻譯」段) |

---

## 長度檢核

| 段落 | 目標字數 |
|------|---------|
| Hook | 80-120 |
| 本週菜單 | 100-150 |
| 每則新聞 × 3-5 | 300-450 each |
| 結尾 | 120-180 |
| **總計** | **1500-2500** |

超過 2500 → 砍冗詞或合併新聞
低於 1500 → 加 1 則或補數據 / 跨連結
